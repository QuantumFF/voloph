# Publish machine analysis as content-keyed files in the shared library

The machine-produced **Analysis** of a recording — the draft timeline with per-rally confidence (uncertain regions included), waveform, duration, and capture day — is published as a small JSON file in a dot-folder at the shared library root (`.voloph/analysis/`), so that any other user of the same library adopts it instead of recomputing it. Analysis is expensive (two full decode passes per recording, ADR 0006/0011) and a pure function of the recording's bytes and the segmenter; before this, the only way it crossed users was inside a session bundle (ADR 0012) — a deliberate, per-sharer act of sharing *review state*, which forced user B to either wait for user A to share or re-burn hours of compute on identical bytes.

The file captures the **pristine machine output at analysis completion, never later corrections**. Hand corrections mutate rallies in place (the pristine draft ceases to exist), and more importantly corrections are review state: carrying them would leak one user's hand-work into another's review without a share action, and would make the recipient look hand-touched — blocking them from silently receiving real bundles. This purity is what licenses every other property: because an Analysis carries no one's judgment, it needs no sharer label, no offer, no keep-mine-or-take-theirs, and no UI. Corrected timelines travel by session bundle, full stop.

Files are **per recording** (partial coverage of a session works naturally) and **content-keyed** — named by quick hash + file size, the recording's existing identity check. The key survives renames and moves within the library, deduplicates copies, and *is* the verification: the adopter computes the quick hash during scan anyway, and either the key matches or there is no entry. Each file carries a `format`/`version` envelope (like `.vbundle`) plus the segmenter version — ignored today, but it lets a future, meaningfully better segmenter offer "re-analyze stale Analyses" without a format change.

**Adoption is silent and conservative.** A newly discovered file with a matching Analysis registers with the carried state — no staging, no probe, no segmentation, playable seconds after scan. A known-but-unanalyzed recording adopts and leaves the queue. An already-analyzed recording is a no-op (the compute is spent; segmenter versions may legitimately disagree). A hand-touched recording is never touched and never even asked. A recording covered by a pending bundle offer adopts the Analysis immediately *and* keeps the offer pending — accepting later applies over machine-only state silently under the existing rule, and declining no longer triggers the expensive analysis.

**Publication is an invariant, not a trigger**: after any scan or analysis pass, every shared-library recording whose analysis in the local DB is still machine-pristine has its file on disk (write-if-absent; unreadable counts as absent; explicit re-analyze is the one overwrite). This makes publish-failure retry, day-one backfill of pre-feature compute, and the local→shared copy journey (ADR 0011 adoption pass) all fall out of one rule. The local library never writes Analysis files — no one else can reach its bytes. Concurrent analyzers last-writer-win harmlessly (both wrote a valid analysis of the same bytes; temp-then-rename keeps half-written files from being adopted).

**Failure is silent, and nothing is ever deleted.** An unreadable, truncated, or wrong-version file is logged and ignored — the normal pipeline analyzes as if it weren't there. This inverts bundle receive's loud refusal deliberately: a bundle was handed over by a human who deserves to know it didn't apply; an Analysis is plumbing the user never knew existed. There is no garbage collection: files are ~KB each, orphans revive if the recording returns, and auto-deleting from *shared* storage is the one destructive cross-user act plumbing must never do. The UI shows nothing new — adoption is indistinguishable from a very fast local analysis.

Chosen over: **extending session bundles** — per-sharer, offered, consent-driven semantics are exactly wrong for an impersonal cache, and bundles require a human to act; **a shared SQLite database at the root** — rejected by ADR 0012 already: whole-file last-writer-wins erases whichever device pushed first; **path-keyed sidecars next to each recording** — visible clutter in users' folders, broken by rename, and still needing hash verification before applying; **publishing the user's current (corrected) timeline** — leaks review state without a share and the pristine draft no longer exists after edits.

This does **not** reverse ADR 0012's "SQLite, not JSON sidecars": SQLite remains the sole working store and source of truth on every device; Analysis files, like bundles, are a transfer format at the library root. It amends ADR 0011 in two places: staging is skipped when *either* a bundle *or* an Analysis carries the work, and pending-bundle recordings are no longer held out of registration — they register from the Analysis and await the offer. The on-disk format is the hard-to-reverse part — these files live in users' shared folders, so format changes must remain readable backward.
