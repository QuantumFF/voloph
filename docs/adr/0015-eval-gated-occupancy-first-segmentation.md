# Eval-gated segmenter replacement: occupancy proposes, motion refines, audio never gates

The hybrid heuristic of ADR 0006 underperforms on real footage. Ranked by observed pain: **missed rallies** (worst — invisible on the timeline, findable only by scrubbing raw footage), false positives (knock-ups and non-play movement kept), merged/shattered rallies, imprecise edges. A minority of recordings carry background noise (a neighbouring court, volleyball thumps), and the prime *suspect* for the misses — to be verified, not assumed — is the **audio confirmation gate**: constant background noise raises the adaptive onset baseline, real shuttle hits stop registering, and the gate silently discards rallies motion had correctly found. ADR 0006 demoted audio to escape neighbour-court bleed; bleed still poisons it from the confirmation side.

**Acceptance bar, asymmetric by error cost.** A missed rally hides play with no marker (confident-but-wrong errors get no uncertain region); every other error is a one-keystroke correction. So: **≈ 0 missed rallies per session** is the constraint; ≤ 5–10 one-keystroke corrections per hour and edges within ~1 s are bounded costs. Recall is the constraint, everything else is optimization — a candidate that misses 3% of rallies is worse than one that over-includes 20% downtime, and no precision is ever bought by paying recall.

**Constraint envelope, revised from ADR 0002/0006.** *Local* stays untouchable (footage never leaves the machine; the shared-library design assumes it). *Replaceable behind the timeline* stays and is what makes this swap contained. *Heuristic-not-learned* may be broken — it was justified by "most of the value at a fraction of the effort," and the value is not being reached. *GPU-free* relaxes to **GPU-optional, CPU-capable**: worst-case analysis ≤ ~1.5× real-time on CPU, GPU (when probed present) only makes it faster, never changes the output.

**Fusion roles, effective immediately (not gated on the detector).** Occupancy proposes candidate play spans; the motion envelope places their edges (its strength was always boundaries, never semantics); **audio never hard-gates again** — it only modulates confidence: hits raise it, a hitless moving span drops it and surfaces as an uncertain region, *but the span stays on the timeline*. Under the zero-miss bar, no single signal may ever delete a rally on its own. Accepted non-goal: a singles **knock-up** is geometrically, kinetically, and acoustically a rally; it stays in the draft (amber at worst) and costs one `delete`. Not to be re-litigated.

**One nano detector, hardware-adaptive, no user choice.** A single small **Apache-2.0-licensed** person detector (YOLOX-nano / NanoDet class — the popular Ultralytics YOLOs are AGPL) shipped as ONNX, run via execution-provider probing (DirectML/CUDA where present, CPU fallback), frames from the existing ffmpeg sidecar at 2–5 fps downscaled. Rejected: a two-model CPU/GPU split with a user decision dialog — the recall bar applies to both models so the pair is either redundant or a violation; the accuracy delta on "find 1–2 large humans on a static court" is ~nil; it doubles tuning and muddies Analysis versioning; and a model-choice dialog fights frictionless import. A bigger model is revisited only if the harness proves the nano *detector itself* (not the logic above it) misses rallies. Rejected outright: end-to-end action recognition (data-hungry, opaque, outside the CPU budget) and cloud inference.

**Geometry-free v1.** No court-line detection, no calibration UI. Static camera behind the court + singles makes bounding-box size a depth proxy: cluster detections per recording and the near/far split falls out; "opposite halves occupied" needs no court quad. The net-side bystander is dropped by staticness (a box that barely moves for minutes is furniture), not geometry. A one-time manual calibration step is an **evidence-gated fallback** — built only if the harness shows recordings (e.g. cut-off near baseline clipping the near player's box) where the statistical split fails. Known cost accepted: only-one-player-visible must never read as "rally over" (the near player leaves frame on deep retreats).

**Eval harness as the referee.** A dev tool (not app UI) scores a candidate segmenter's draft against fully hand-corrected timelines: missed rallies (target 0), false positives per hour, median boundary error. Gold corpus: the current ~5 hours of footage, fully corrected — including the noisy recordings. The corpus doubles as tuning data, so corpus numbers are evidence, not proof; new footage stays the real test. Hand-corrected rallies are already distinguishable in the DB (confidence sentinel) and the pristine draft survives in the Analysis file (ADR 0013), so gold and machine output are separable artifacts.

**Three stages; the detector is conditional.** Stage 0: instrument the two kill-gates (motion never fired vs. audio rejected) with per-span verdicts, correct the corpus, build the harness. Stage 1: demote the audio gate per the fusion rules plus whatever Stage 0's diagnosis directly indicates; re-measure. Stage 2: the detector — **built only if Stage 1's numbers miss the bar**. Forecast recorded honestly: Stage 1 likely fixes the misses but worsens false positives (spans audio over-zealously rejected now survive), and occupancy geometry is the only mechanism here that attacks the expensive false positives (shuttle-collecting, milling-about) — so Stage 2 probably happens, but on evidence rather than on the assumption that "computer vision" was the answer.

**Staleness rules** (Stage 1 already "materially alters the draft," the `SEGMENTER_VERSION` bump trigger of ADR 0013): a hand-touched timeline is never clobbered by any version bump — corrections are the most authoritative data in the system; explicit per-recording Re-analyze remains the one override. Untouched recordings re-analyze automatically and silently when the active segmenter outranks their Analysis. A stale shared-library Analysis is superseded (publish alongside, adoption prefers the highest version), never deleted. Bundles carry hand-corrected state and are untouched by all of this. Consequence stated out loud: since the whole existing library will be hand-corrected for the corpus, the improved segmenter is only ever *visible* on newly imported recordings — that is correct, not "the upgrade did nothing."

Partially supersedes ADR 0006 (audio demoted from hard gate to confidence modulator; a learned detector may become the semantic primary) and further revises ADR 0002 (GPU-free and heuristic-not-learned relaxed). Local, static-camera assumption, inclusion bias, per-region confidence, and replaceable-behind-the-timeline all still hold.
