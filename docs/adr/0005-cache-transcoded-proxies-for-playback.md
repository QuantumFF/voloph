# Cache transcoded proxies for playback

Recordings the webview cannot decode (notably iPhone HEVC, which WebKitGTK refuses) are made playable by transcoding each one **once** to a cached, seekable H.264/AAC **proxy** stored in the app data dir, played through the Tauri asset protocol — rather than transcoding live on every play and streaming the result.

Chosen because the review player is **seek-dominated**: gap-free playback auto-skips between rallies, the user scrubs the timeline strip, and inline correction nudges rally boundaries frame by frame. Live transcoding cannot serve this — it has no seekable byte layout, so a seek means killing ffmpeg and relaunching it with `-ss` (~1s, keyframe-accurate only), and fragmented `empty_moov` output has no known duration for the timeline strip. Every gap-skip would be a relaunch: the opposite of zero-downtime. A cached complete file gives native frame-accurate seeking and a real duration, and reuses the asset-protocol path that already plays H.264 untouched. Originals that are already H.264/AAC need no proxy and play directly.

Proxies are generated **eagerly in the background at import**, alongside audio segmentation (see ADR 0002), so every proxy in a session is ready before the user opens it to review — protecting gap-free playback. This is justified because the user imports their own games specifically to watch them, so every imported recording is intended for review.

Constraints to respect, not visible in code: a proxy costs disk (a full re-encode per incompatible recording) and CPU at import. A proxy is **derived and rebuildable**, never the source of truth — keyed to the recording so a deleted or stale proxy is simply regenerated, and deleting the cache must always be safe. Originals are never touched (ADRs 0001, 0003). Cache-size management (eviction, a budget) is out of scope for now; the cache only grows.

Supersedes the live transcode-and-stream playback approach described in ADR 0004 and implemented in issue #15 (commits `63ecc50`, `01e46eb`); that loopback streaming server, chunked transfer, and seek-restart logic are removed. ADR 0004's decision to bundle ffmpeg as a sidecar still stands — proxy generation and codec probing use it.
