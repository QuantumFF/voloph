# Occupancy reads depth from box size and proposes by windowed firing density

The Stage 2 occupancy fusion as first built (ADR 0015, issue #84) proposed **zero spans on every real recording at every knob setting** — the #85 measurement checkpoint on #75 traced all 49 missed gold rallies to fusion logic muting a signal the nano detector was delivering. Three structural defects, each refuted by measurement: the opposite-half test wanted a vertical-center gap that this camera height never produces (every box sits at cy ≈ 0.63 ± 0.02, while in-rally box *areas* differ by median 4.4×); the near/far split — midpoint of (min, max) box area — was destroyed by a single near-camera passer-by (area 0.21–0.50 vs player median ~0.02); and gates aggregated over maximal runs of *occupied* samples, which span 85–655 s when someone is visible 94–99% of the time, leaving no operating point between propose-nothing and propose-everything. This ADR replaces the *mechanism*; issue #91 carries the evidence and the revision contract.

**Depth is read from box size, never frame height.** A sample shows two-player structure when it holds two plausible boxes whose area ratio reaches `occupancy_ratio`. The vertical-center test and its `MIN_HALF_SEPARATION` constant are deleted; "opposite halves" retires as a concept — on a low camera behind the court, near/far is a size statement, and the harness showed the vertical branch identically false on all measurable footage. A per-recording choice of separation axis (vertical vs size) was rejected as untestable against the current corpus; it returns only as an evidence-gated fallback if future footage demands it.

**Plausibility by robust statistics, not extremes.** Boxes larger than `occupancy_area_cap_k` × the recording's **median** surviving-box area are discarded before pairing — a near-camera passer-by cannot move a median the way it moved the min/max midpoint. The staticness filter (ADR 0015: bystanders die by staticness, not geometry) is unchanged and runs first.

**Proposals are judged as windowed firing density on the block grid, mirroring motion's active mask.** A sample *fires* when it shows two-player size structure **and** inter-sample movement (some surviving box center moved ≥ `occupancy_static_frac` since the previous sample — the same constant that defines furniture, doubling as the speed floor). A block is occupancy-proposed when the firing density in a `occupancy_window_ms` window at its center reaches `occupancy_density`. The occupied-run span unit is deleted, and with it `occupancy_active_frac` and `occupancy_opposite_frac`. Density below 1 plus the existing bridge absorb one-player-visible stretches, so only-one-visible still never reads as rally-over. Downstream (union with motion, run/bridge/min-rally/pad) is untouched: occupancy only ever adds blocks, never deletes a motion rally.

**ADR 0015's principle stands; only its mechanism paragraph is superseded.** Still true and reaffirmed: geometry-free v1 (no court-line detection, no calibration UI), bystanders dropped by staticness, box size as the depth proxy under the static-camera assumption, the evidence-gated one-time-calibration fallback, the zero-miss degradation rule (absent/failed detector ⇒ exactly the motion-proposes path), and the eval harness as referee. Superseded: the opposite-half vertical test, the min/max area split, and the occupied-run judging unit.

**Measured consequence, recorded honestly (issue #91, segmenter v5).** The rebuilt proposal is alive — misses fall 49 → 22 on the gold corpus (one recording to zero) at 58.3 FP/h (bound ≤ 65) — but the full sweep shows a monotone trade: every further miss rescued costs edge quality, because between-rally firing (~32% of samples) sits too close to in-rally firing at 3 fps for the density judge to separate cleanly; median boundary error at the chosen defaults is 1.50 s (v3: 0.81 s), and the ≈ 0-miss bar is reachable only in a propose-everything regime with tens-of-seconds edges. The residual misses are short rallies (mostly 3–5 s ≈ 9–15 detector samples) whose in-rally firing never clears the density threshold. Closing that gap needs a stronger per-sample signal (detector sample rate, rule refinement), not knob values — the same shape of finding that produced this ADR.
